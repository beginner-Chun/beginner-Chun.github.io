SpringBoot简介

SpringBoot入门

  1. 创建新模块，选择spring初始化，并配置模块相关基础信息

  3. 运行控制器类

SpringBoot项目快速启动

  1. 对SpringBoot项目打包(执行Maven构建指令package)
  2. 执行启动指令
    1. java -jar springboot项目名.jar

SpringBoot概述

  1. 起步依赖

    2. SpringBoot在创建项目时，采用jar的打包方式
    3. SpringBoot的引导类是项目的入口，运行main方法就可以启动项目。

  2. 作用：
    1. 1级与2级留作系统打包后设置通用属性
    2. 3级与4级用于系统开发阶段设置通用属性

自动配置原理

  1. 在主启动类上添加了SpringBootApplication注解，这个注解组合了EnableAutoConfiguration注解。
  2. EnableAutoConfiguration注解又组合了import注解，导入了AutoConfigurationSelector类
  3. 实现selectIpmorts方法，这个方法经过层层调用，最终会读取META-INF目录下的后缀名为imports的文件，boot2.7以前的版本，读取的是spring.factories文件。
  4. 当前到全类名之后，会解析注册条件，也就是@Conditional及其衍生注解，把满足注册条件的bean对象自动注入到IOC容器中。

标准数据层开发

lombok

  1. lombok,应该java类库，提供了一组注解，简化pojo实体类开发
    2. @Data：为当前实体类在编译期设置对应的get、set方法，无参、有参构造方法，toString方法，hashCode方法，equals方法

分页功能

  1. 设置分页拦截器作为spring管理的bean
  2. 执行分页查询
  3. 开启日志

条件查询

  1. 常规格式
  2. lambda格式
  3. 组合条件查询
  4. null值处理

查询投影

  1. 查询结果包含模型类中部分属性
  2. 查询结果包含模型类中未定义的属性

查询条件

  1. eq匹配
  2. le ge匹配 或 between匹配
  3. like匹配
  4. 分组查询聚合函数

字段映射与表名映射

  1. @TableField
  2. 作用：设置当前属性对应的数据库表中的字段关系
  6. 相关属性
    1. value：设置数据库表字段名称
    2. exist：设置属性在数据库表字段中是否存在，默认为true。此属性无法与value合并使用
    3. select：设置属性是否参与查询，此属性与select()映射配置不冲突
  1. @TableName
  2. 作用：设置当前类对应于数据库表关系
  4. 相关属性
    1. value：设置数据库表名称

id生成策略控制

  1. @TableId
  2. 作用：设置当前类中主键属性的生成策略
  4. 相关属性
    1. value：设置数据库主键名称
    2. type：设置主键属性的生成策略，值参照IdType枚举值
  6. 全局配置

多记录操作

  1. 按照主键删除多条记录
  2. 根据组件查询多条记录

逻辑删除

  1. 定义：为数据设置是否可用状态字段，删除时设置这段为不可用状态，数据保留在数据库中。
  2. 步骤
    1. 数据库表中添加逻辑删除标记字段
    2. 实体类中添加对应字段，并设定当前字段为逻辑删除标记字段
    3. 配置逻辑删除字面值

乐观锁

  1. 数据库表中添加锁标记字段
  2. 实体类中添加对应字段，并设定当前字段为乐观锁标记这段
  3. 配置乐观锁拦截器实现锁机制对应的动态SQL语句拼装
  4. 使用乐观锁机制在修改前必须先获取对应数据的version方法方可正常进行
